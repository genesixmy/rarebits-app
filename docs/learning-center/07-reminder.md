## Tujuan Modul Reminder

Reminder membantu disiplin operasi:
- follow-up pelanggan,
- restock,
- urusan event,
- tugasan admin bisnes.

## Fungsi Utama

1. Tambah reminder baru
- Tajuk, tarikh mula, kategori, priority, nota.

2. Recurrence
- None, daily, weekly, monthly.
- Boleh tetapkan interval dan tarikh tamat.

3. View mode
- List view untuk tindakan segera.
- Calendar view untuk perancangan jadual.

4. Completion tracking
- Tanda reminder siap.
- Untuk recurring reminder, completion ikut occurrence.

## Cara Guna Praktikal

1. Cipta reminder untuk setiap komitmen yang boleh terlepas.
2. Pilih kategori supaya senang tapis.
3. Guna priority tinggi hanya untuk perkara benar-benar kritikal.
4. Set recurring untuk tugasan rutin mingguan/bulanan.

## Cadangan Setup Kategori Reminder

Contoh kategori praktikal:
1. Ops
2. Payment
3. Event
4. Restock
5. Customer
6. General

## Rutin Harian Dengan Reminder

1. Mula hari: semak reminder due/overdue.
2. Selepas selesai: terus mark complete.
3. Akhir hari: semak baki reminder belum selesai.

## Best Practice

1. Elak tajuk reminder terlalu umum.
2. Gunakan recurrence berbanding cipta reminder baru berulang kali.
3. Simpan nota ringkas konteks kerja supaya mudah disambung.
