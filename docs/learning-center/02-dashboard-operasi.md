## Tujuan Dashboard

Dashboard ialah ringkasan keadaan bisnes semasa untuk bantu keputusan cepat:
- jualan,
- keuntungan,
- caj platform,
- refund/pelarasan,
- pecahan platform/kategori.

Secara default, paparan fokus tempoh terkini (rolling window) supaya trend operasi nampak terus.

## Komponen Utama Yang Perlu Difahami

1. Ringkasan metrik
- Revenue, net profit, fee, refund, expense.

2. Graf pecahan jualan
- Pecahan ikut platform atau kategori.
- Diguna untuk nilai channel mana paling aktif.

3. Insight operasi
- Isyarat risiko stok/aging.
- Data pending operasi (jika ada).

4. Reminder ringkas (jika dipaparkan)
- Susulan tugas penting harian.

## Cara Guna Dashboard Dengan Betul

1. Mula dengan tempoh tarikh yang relevan
- Semak 30 hari untuk health semasa.
- Semak 7 hari untuk tindakan cepat.

2. Semak net profit sebelum buat keputusan belian stok
- Elak tambah stok jika margin semasa menurun.

3. Bandingkan pecahan platform
- Jika satu platform terlalu dominan, pertimbang risiko channel.

4. Semak perubahan luar biasa
- Lonjakan refund/caj platform biasanya tanda data operasi perlu audit.

## Tafsiran Ringkas Metrik

1. Revenue naik, net profit turun
- Selalunya sebab caj platform, diskaun, refund, atau kos item naik.

2. Jualan item banyak, cashflow ketat
- Semak wallet: mungkin banyak expense belum seimbang.

3. Platform tertentu jual banyak tetapi margin rendah
- Semak rule caj platform dan strategi harga untuk platform itu.

## Best Practice Dashboard

1. Lihat dashboard selepas tutup operasi harian.
2. Gunakan angka dashboard untuk keputusan, bukan memory.
3. Jangan ubah data lama tanpa sebab jelas.
4. Simpan snapshot/backup berkala untuk audit dalaman.
